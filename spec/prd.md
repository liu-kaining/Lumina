# 💎 Lumina 产品需求文档 (PRD) v1.0

## 一、 产品概述 (Product Overview)

* **产品名称**：Lumina (光影档案)
* **产品形态**：Google Chrome 扩展程序 (基于 Manifest V3)
* **核心价值**：打破“文本编辑”与“AI 配图”的工具壁垒。将网页划选的上下文片段，通过大模型（Gemini 3 Flash + Nano Banana 2）的语义重构，无缝转化为高品质、风格统一的本地化视觉资产。
* **目标用户**：内容创作者（自媒体、博主）、新媒体运营、撰稿人、PPT 报告制作者。

---

## 二、 核心用户心智流 (User Flow)

1. **发现与捕捉**：用户在网页（如 Notion）写作时，划选一段描述性文字。
2. **轻量唤起**：光标上方出现悬浮的 Lumina Logo。点击（或使用快捷键 `Cmd+Shift+L`）唤起右侧 SidePanel。
3. **语义重构**：Lumina 提取“选中文字 + 网页标题 + 上下文”，静默调用大模型生成专业绘画指令。
4. **渲染展示**：在极简的毛玻璃侧边栏中，展示生成中的流光动效，随后图片渐显。
5. **消费资产**：用户直接将图片拖拽进原网页编辑器，或点击“打包”导出当前文档的所有配图。

## 三、 详细功能需求说明 (Detailed Functional Requirements)

### 模块 1：多维触发机制 (Entry Points)

为了保证“无感”，我们提供三种触发方式，涵盖不同使用习惯。

| 触发方式 | 触发条件 | 交互表现与规则 |
| --- | --- | --- |
| **划选悬浮标 (Tooltip)** | 用户使用鼠标划选网页文本，且字符数 > 2。 | 在选区正上方 10px 处，弹出带有弹性动画 (Spring) 的 Lumina Logo。点击即打开/更新 SidePanel 并开始生图流程。点击页面空白处该图标消失。 |
| **右键菜单 (Context Menu)** | 选中文本后，右键点击。 | 菜单中新增一项：“✨ 发送至 Lumina 生成配图”。 |
| **全局快捷键 (Shortcut)** | 选中文本后，按下 `Cmd/Ctrl + Shift + L`。 | 直接强制打开 SidePanel，并吞入选中文本执行生成。 |

### 模块 2：智能侧边栏工作区 (The SidePanel Workspace)

这是整个产品的 UI 核心，分为三个区域。

#### 2.1 顶部：状态与配置区 (Header & Settings)

* **全局预设开关**：一个精致的下拉胶囊按钮，选择默认绘画风格。
* *预设列表*：Default (智能推断), Cinematic (电影感), Isometric 3D (等距3D), Minimalist Flat (极简矢量), Ghibli Style (吉卜力风)。


* **API 设置入口**：点击齿轮图标，翻转进入设置页，输入 Gemini API Key。**规则**：Key 只存本地，不显示明文（显示为 `sk-...xxxx`）。

#### 2.2 中部：创作流面板 (Creation Canvas)

* **原文卡片**：展示当前抓取到的文字。如果文字过长，最多显示 3 行，尾部显示 `...`，悬停显示全文本。
* **Prompt 棱镜 (Lumina Prism)**：
* **机制**：默认隐藏，点击“✨ 展开 Prompt”后显示 AI 增强后的英文绘图指令。
* **交互**：允许用户直接在这个文本框里修改单词，修改后按下 Enter 可直接触发重新生成。


* **生图画框 (The Frame)**：
* **Loading 状态**：不允许使用转圈圈！必须使用骨架屏 + 背景微光扫过 (Shimmer) 的动画。
* **Success 状态**：图片加载完毕后，使用 0.3s 的 Fade-in 渐显。图片支持 CSS `object-fit: cover`，保持 16:9 比例展示。
* **操作按钮 (悬停在图片上时浮现)**：
* `拖拽把手`：提示用户可 Drag & Drop。
* `重新生成 (Re-roll)`：使用相同参数重新请求一次。
* `单图下载`：直接触发浏览器下载。





#### 2.3 底部：光影画廊 (Lumina Archive)

* **历史记录 (瀑布流)**：展示当前会话中（或当前网页 URL 下）生成的所有历史图片缩略图。
* **一键打包 (Bundle Export)**：
* 点击“打包下载 (ZIP)”，前端利用 JSZip 将该列表内的所有图片打包。
* **命名规则**：`Lumina_[日期]_[网页标题]/[序号]_[原文前4个字].png`。

## 四、 核心业务逻辑与规则 (Business Logic & Rules)

### 4.1 "上下文感知" 抓取规则

简单的生图工具只抓选中的字。Lumina 必须抓取上下文，以提高生成的语义连贯性。

* **抓取目标 1 (必选)**：`window.getSelection().toString()` (用户划选的字)。
* **抓取目标 2 (隐式)**：`document.title` (当前网页标题)。
* **抓取目标 3 (隐式)**：选中文本所在 DOM 节点的前后各 100 个字符。
* *组合策略*：发送给 Gemini 增强 Prompt 时，系统提示词应写明：“用户划选的短语是 [目标1]，它所在的网页主题是 [目标2]，前后文是 [目标3]。请结合语境，将其转化为一幅画的描述。”

### 4.2 状态机与错误处理 (State Machine & Edge Cases)

这是开发最容易漏掉的地方，PRD 必须定死：

| 异常场景 / 边界情况 | Lumina 的 UI 表现与处理规则 |
| --- | --- |
| **未配置 API Key** | SidePanel 呈现“磨砂玻璃锁定”状态，中心显示一个大号输入框要求填 Key。填完后才解锁界面。 |
| **选中文本过短/过长** | **< 2 字符**：不弹出悬浮标。**> 500 字符**：悬浮标照常弹出，但在 SidePanel 中提示“文本过长，Lumina 已自动提取核心意图”，并交给大模型做摘要后再绘图。 |
| **API 请求超时 / 失败** | 画框内显示红色的“断裂光束”图标，附带简短的非技术化提示（如：“灵感连接中断，请检查网络或重试”），提供一键重试按钮。 |
| **触发安全审核 (NSFW)** | Nano Banana 模型返回安全拦截时，显示“该描述包含受限内容，Lumina 无法具象化”。 |
| **用户关闭了 SidePanel** | 正在生成的任务不应中断（Service Worker 在后台继续）。用户再次打开 SidePanel 时，应能看到刚才生成的图片已在画廊中。 |

## 五、 数据与隐私安全规范 (Data & Privacy Specs)

Lumina 的核心卖点是**“本地化工作流”**，这一点要在架构设计中强制体现：

1. **零遥测 (Zero Telemetry)**：绝对不允许引入 Google Analytics 或任何用户行为追踪 SDK。
2. **纯本地存储 (Local First)**：
* API Key 采用 `chrome.storage.local` 存储，插件卸载即销毁。
* 生成的图片数据，API 返回后必须立即转化为 `Blob` 对象，存入 `IndexedDB` (通过 Dexie.js 操作)。严禁将图片长久保存在内存中的 Base64 变量里（会导致浏览器崩溃）。


3. **网络请求白名单**：所有的 Fetch 请求，仅允许指向 `https://generativelanguage.googleapis.com` (Gemini API 官方端点)。

---

## 六、 性能与指标 (Performance Metrics)

* **启动响应**：划选文本到 SidePanel 开始响应，延迟 `< 100ms`。
* **Prompt 增强**：Gemini 3 Flash 处理时间应控制在 `< 1.5s`。
* **图片生成**：依赖 Nano Banana 2 的响应速度，前端必须在长等待期间提供高质量的视觉安抚（粒子动画或流光动画）。
* **内存占用**：通过定期清理老旧 Blob 缓存，确保插件闲置内存占用 `< 50MB`。
